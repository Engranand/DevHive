// How It Works

One PR merge. Watch the
whole system react.

This single sequence explains what makes DevHive different — everything connected, nothing manual.

1
PR merged on GitHub
Rohan merges PR #142 — "feat(auth): oauth callback" on the GitHub repo.
2
Webhook fires instantly
GitHub sends a pull_request.closed event to the DevHive Express webhook handler.
3
Task auto-moves to Done
The linked task moves Review → Done via Socket.io broadcast to all 4 online members simultaneously.
4
Workload recalculated
BullMQ triggers a background workload job. Anand still shows 42% above team average.
5
AI alert fires
The engine emits an imbalance event. Team lead gets a reassignment suggestion in real-time — no refresh needed.

// FAQ

Common questions.

How does the AI workload engine actually work? +
Score = Σ (priority_weight × urgency). Critical=4x, High=3x, Medium=2x, Low=1x. Urgency = 1 / days_until_due. If any member exceeds team average by 40%, an alert fires. BullMQ runs this every 30 minutes as a background job — zero manual intervention needed.
How is DevHive different from Jira or Linear? +
Jira and Linear are built for enterprise teams of 50+. DevHive is for 2–10 person teams who want real power without the complexity. The AI workload fairness engine and Hive Board (team discovery at idea stage) don't exist in any other tool.
What happens exactly when a GitHub PR is merged? +
If you've linked a task to that PR, DevHive automatically moves the task to "Done" via a GitHub webhook — triggered on pull_request.closed with merged=true. All team members see the update instantly via Socket.io. Workload recalculation is also queued automatically.
Can I self-host DevHive? +
Yes. DevHive is open source (MIT license). You need Node.js v18+, MongoDB, and Redis. Clone the repo, fill your .env with the required keys, and run npm run dev. Full deployment guide with Render + Vercel setup is in the README.
What is the Hive Board exactly? +
Hive Board is a public feed where developers post project ideas and recruit teammates. Unlike GitHub (only shows existing repos), Hive Board helps teams form at the idea stage — before a single commit is made. Post your idea, specify what roles you need, review join requests, and build together inside DevHive.
